Git HTTP协议简介

从git官方文档上看，git实现http(s)协议有两种方式

  1. 智能（Smart）HTTP
  2. 哑（Dump）HTTP

其中Smart HTTP是自Git 1.6.6版本开始引入的一种新的、更智能的协议。基于该协议可以实现对git仓库的读写功能。而在Dump HTTP里 web 服务器仅把裸版本库当作普通文件来对待，提供文件服务,更多的是用来实现只读功能。
